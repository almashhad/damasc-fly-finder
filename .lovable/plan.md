
# اعادة تصميم صفحة نتائج البحث - بسيطة ونظيفة

## المشكلة الحالية
صفحة البحث (`/search`) تحتوي على:
- محرك بحث كامل (FlightSearchForm) يظهر في الاعلى - غير ضروري بعد البحث
- قسم فلاتر جانبي معقد (sidebar) مع checkboxes
- بطاقات الرحلات كبيرة ومعقدة (FlightCard) مع اقسام متعددة
- Layout مع Header و Footer اضافي

## التصميم الجديد المقترح
صفحة نتائج بسيطة جدا بنفس اسلوب الصفحة الرئيسية (Index.css style)، تعرض فقط الرحلات مع فلتر بسيط في الاعلى.

### هيكل الصفحة

```text
+------------------------------------------+
|  →  دمشق ← دبي     |   6 رحلات متاحة   |
+------------------------------------------+
| الأرخص | الأقصر | وقت المغادرة | مباشرة  |
+------------------------------------------+
|                                          |
| [بطاقة رحلة بسيطة]                      |
| +--------------------------------------+ |
| | أجنحة الشام  SAW301                  | |
| |                                      | |
| | 11:00  ----✈---  13:30    $165      | |
| | دبي DXB        دمشق DAM   مباشرة   | |
| |              2س 30د                  | |
| +--------------------------------------+ |
|                                          |
| [بطاقة رحلة اخرى...]                   |
+------------------------------------------+
```

- نفس الالوان: primary blue `hsl(217 91% 60%)`
- نفس الحدود: `hsl(220 13% 91%)`
- نفس الخلفية: ابيض نقي
- نفس الـ border-radius: 16px للبطاقات
- نفس الخطوط: Rubik
- متوافق مع الموبايل (breakpoints عند 480px و 360px)

### بطاقة الرحلة البسيطة (ضمن Search.css)

```text
+------------------------------------------+
| [شعار] أجنحة الشام    SAW301            |
|                                          |
|  11:00     ----✈----     13:30           |
|  دبي DXB    2س 30د     دمشق DAM        |
|             مباشرة                       |
|                                          |
|  $165               [احجز →]            |
+------------------------------------------+
```

### منطق الفلترة

- **الارخص**: ترتيب حسب `price_usd` تصاعديا (الافتراضي)
- **الاقصر**: ترتيب حسب `duration_minutes` تصاعديا
- **وقت المغادرة**: ترتيب حسب `departure_time`
- **مباشرة فقط**: تصفية `stops === 0` فقط (toggle)

### URL Parameters المدعومة

- `type`: `to_damascus` / `from_damascus` / `to_aleppo` / `from_aleppo`
- `airport`: `DAM` / `ALP` (لتحديد المطار المحلي)
- `destination`: كود المطار البعيد (مثلا `DXB`)
- `passengers`: عدد المسافرين
- `date`: تاريخ السفر
